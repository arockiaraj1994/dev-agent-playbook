1. Add the server

Run once in your terminal — this writes to your user-level Claude Code config:

claude mcp add {{ server_label }} --transport sse {{ sse_url }}

2. With authentication

If the server requires a token, pass it as a header:

claude mcp add {{ server_label }} --transport sse \
  --header "Authorization: Bearer {{ token_val }}" \
  {{ sse_url }}

3. Project-scoped install (optional)

To make the server available only inside one repository, run the same command from inside the repo with --scope project:

cd ~/code/my-app
claude mcp add {{ server_label }} --transport sse --scope project \
  --header "Authorization: Bearer {{ token_val }}" \
  {{ sse_url }}

Add project rules

Rules, architecture docs, and skills are markdown files indexed for search. Anyone on your team can add them — here's how.

1. Choose a namespace

Drop your file under one of three folders in the rules-repo:

  • rules/ — coding standards, conventions, "do this not that" guidance
  • arch/ — architecture overviews, system designs, ADRs
  • skills/ — runbooks, checklists, multi-step procedures

2. Write the doc

Add a frontmatter block with at minimum title and tags. Tags weight the search index heavily — use the words your team would actually type.

---
title: API error handling
tags: [api, errors, http, status-codes]
projects: [payments-svc, api-gateway]
---

# API error handling

When a request fails, return a JSON envelope:

```json
{ "error": { "code": "RATE_LIMITED", "message": "..." } }
```

...

3. Open a PR

Push the file, open a PR against main. The CI bot will:

  1. Lint frontmatter (title, tags required)
  2. Re-index the SQLite FTS table
  3. Hot-reload the server (no restart needed)
Watch the Searches → Zero-result tab — every entry there is a doc your team is asking for. Pick one, write it, and you'll see the badge clear in real time.

4. Pin to specific projects (optional)

Use the projects: frontmatter array to scope a doc. Calls from list_projects → search_rules(project=…) will return scoped results first.

Verify it works

  1. Restart your editor after adding the MCP server.
  2. Ask your AI: "list available projects" — you should see the project names returned from this server.
  3. Ask your AI: "search rules for error handling" — you should see relevant rule doc results inlined.
  4. Open the Users page — your username should appear with an active badge after your first call.
